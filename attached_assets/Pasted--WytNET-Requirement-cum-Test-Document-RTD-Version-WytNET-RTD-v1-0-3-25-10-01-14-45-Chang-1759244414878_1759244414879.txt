Initial Phase 1 RTD Draft.

1. Clarifications (as per Replit Agent Feedback)
1.1 Database Platform

RTD v1.0.2 said: “All database via Supabase.”

Reality: Current system uses Neon PostgreSQL + Drizzle ORM.

Decision: ✅ We will continue with Neon PostgreSQL + Drizzle ORM (stable, zero risk).

Supabase optional in future for storage/realtime features only.

1.2 Authentication (WytPass)

RTD v1.0.2 said: “Use WytPass Auth system.”

Reality: Current custom auth already exists (Google OAuth, Email/Password, OTP).

Decision: ✅ “WytPass” is branding for existing auth system.

No separate new auth system required, only ensure consistent branding.

1.3 Admin Infrastructure

Question: Extend existing /admin or build separate backend?

Decision: ✅ Extend existing /admin routes & infra.

Reason: Already role-based guards + CMS present, risk is low.

1.4 Priority Order for Phase 1 Features

Based on feasibility and foundation-first approach:

Phase 1A (Foundation)

Add WytPoints schema (wallet + ledger).

Extend Razorpay flow for recharge + app purchases.

Entitlements table for app access.

Show WytPoints balance in admin + user dashboard.

Phase 1B (Features)

WytApps catalog (per-output fees, subscriptions).

Automate WytPoints earn/spend rules.

Build WytLife landing page /wytlife + application form system.

Phase 1C (Polish)

Content & Stream → migrate static pages into DB.

Add AI-assisted content generation.

Add rich media (images, animations, video).

Add Preview & Publish workflow.
