## 1. Public Website Homepage (Visitor View)

**URL:** `wytnet.com/`

### Layout (3-Column Grid)

* **Left Bar:**

  * Minimal filters: Category, Location.
  * Quick links disabled → show “Login to access MyNeeds/MyOffers”.

* **Center (WytWall Stream):**

  * Show only **Public Needs** (title, short desc).
  * Tabs: All Stream | Jobs | Real Estate | B2B Supply | Service.
  * Offer button disabled → shows **“Login to Offer”**.
  * Sponsored Needs pinned at top.

* **Right Bar:**

  * WytApps preview grid (Duty, Task, Goals, etc.) → read-only.
  * Promotions:

    * “Join WytNet → Post Needs & Earn Points.”
    * “Apply for WytLife.”

### Purpose

* Public visitors see **live activity (Needs)** → platform looks active.
* Conversion funnel: CTA to **Signup/Login**.

## 3. WytStar Program (Community Contribution & Recognition)

* **Brand:** WytStar 🌟
* **Purpose:** Build verified datasets via community contributions.
* **Contribution types:** Post Needs, Offers, Listings, add images/details.
* **Verification:** Community verifiers confirm entries.
* **Rewards:** Contributor & Verifier both earn WytPoints.

### Star Levels

1. Bronze (0–199 pts)
2. Silver (200–999 pts)
3. Gold (1000–4999 pts)
4. Platinum (5000–14999 pts)
5. Diamond (15000+)

### UI

* Profile badge (🌟 Bronze → 💎 Diamond).
* Leaderboard = Hall of WytStars.

### Admin Control

* Configure point values per action.
* Verification thresholds.
* Monthly bonuses & caps.

---

## 4. Profile Completion Economy

* **Welcome Bonus:** +10 pts.

* **Profile Fields → Reward:**

  * Basic info (Name, Email, Phone, City) = +5 pts.
  * Demographics (Age, Occupation, Interests) = +5 pts.
  * Upload photo/cover = +5 pts.
  * First Need posted = +10 pts.
  * First Offer posted = +10 pts.

* **Mandatory Rule:**

  * Profile completion >=70% required for full platform access.
  * 100% completion required to spend points / unlock matches.

* **UI:**

  * Progress bar (“Your profile is 40% complete. Finish now to earn +30 pts”).
  * “Complete Profile → Unlock Matches & Offers.”

## 5. WytMatch (Matchmaking Stream)

* **New Menu Tab:** WytMatch (after login).
* **Logic:**

  * MyNeeds ↔ Other Offers.
  * MyOffers ↔ Other Needs.
* **UI:**

  * Match cards: “3 Offers match your Need: Office Chair.”
  * CTA → “Unlock Match (Cost: 1 pt)”.
* **Rules:**

  * Daily free quota (eg. 3 free matches).
  * Extra unlocks cost points.
  * Streak bonus & WytStar level = more free matches.

## 8. UI/UX Guardrails – Standard Items

### Responsive Design

* All panels (Public, User, Org, Admin) must be mobile-first responsive.
* Support breakpoints: Desktop, Tablet, Mobile.
* Ensure PWA support for mobile.
* Sidebar collapses on mobile; tables collapse to cards.

### Dark Mode & Light Mode

* Must have system-wide theme toggle.
* Default = follow system settings.
* All components must support theme switch.

### Panels Consistency

* Same base layout grid (Header + Sidebar + Content).
* Role-based menus inside Sidebar.
* Seamless switch between panels.

### Shared Components

* Header, Sidebar, Footer = single source of truth.
* Auth/role-based rendering only.

### Accessibility Standards

* All buttons, links = keyboard accessible.
* Screen reader labels.
* Font scaling support.

### Localization

* Text from localization file.
* Default: English, future Tamil/Hindi support.

### Performance

* Pages must load <2 sec on 4G.
* Lazy loading for images.
* CDN for static resources.

### Branding

* WytNet logo top-left.
* Brand mantra: Speed ⚡ | Security 🔒 | Scale 📈.
* WytStar badges + WytPoints visible consistently.

## 9. Development Philosophy & Guardrails

* **Speed** 🚀 – Fast actions, minimal steps.
* **Security** 🔒 – Strong auth, verified data, safe payments.
* **Scale** 📈 – Design for 1M+ users, plug-and-play apps.

**Rules for Developers:**

1. New features must be additive, not destructive.
2. Existing working flows must remain intact.
3. Enhancements = overlay, not replace.
4. Each RTD version must follow change log exactly.

## 10. Shared Components Policy (Auth-aware)

* Header, Sidebar, Footer to be created once under `/components`.
* Must support authentication check:

  * Before auth → show Login/Join.
  * After auth → show User Menu, Wallet, Notifications.
* Sidebar adapts to role: User, Org, Admin.
* Layout wrappers ensure consistent usage across all pages.
