RealBro Property Brother - Module Integration Requirements for WytNet Multi SaaS
Overview
Integrate the RealBro Property Brother application as a standalone module/app within the WytNet Multi SaaS platform. This is a Tamil-focused real estate management PWA designed for property brokers in Tamil Nadu, India.

Core Application Details
Technology Stack
Frontend: Pure JavaScript (React via CDN), no build process required
Architecture: Single Page Application (SPA) with client-side routing
Storage: localStorage for data persistence (production would use backend DB)
Styling: Inline styles with responsive design
PWA: Service worker enabled with offline support
Key Features to Integrate
Authentication System: Username/Password + SMS OTP via MSG91
Property Management: CRUD operations with credit-based listing system
Contact Management: Broker network management
Credit Wallet: Razorpay payment integration for property listing credits
Admin Panel: System administration with user management
PWA Features: Installable app with offline support
Location Services: Mappls API integration for property mapping

3. Database Schema (for production backend)
-- Users table
CREATE TABLE realbro_users (
  id VARCHAR(10) PRIMARY KEY,
  username VARCHAR(50) UNIQUE,
  password_hash VARCHAR(255),
  name VARCHAR(100),
  phone VARCHAR(15),
  email VARCHAR(100),
  district VARCHAR(50),
  role ENUM('broker', 'admin'),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  last_login TIMESTAMP,
  is_demo BOOLEAN DEFAULT false
);
-- Properties table
CREATE TABLE realbro_properties (
  id VARCHAR(20) PRIMARY KEY,
  user_id VARCHAR(10),
  title VARCHAR(100),
  size_text VARCHAR(50),
  price_max INT,
  price_min INT,
  commission_type ENUM('PERCENT', 'FIXED'),
  commission_value DECIMAL(10,2),
  lat DECIMAL(10,6),
  lng DECIMAL(10,6),
  status ENUM('AVAILABLE', 'ON_HOLD', 'SOLD'),
  share_slug VARCHAR(50),
  title_slug VARCHAR(100),
  notes TEXT,
  photos JSON,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  credit_used BOOLEAN DEFAULT true,
  FOREIGN KEY (user_id) REFERENCES realbro_users(id)
);
-- Credits table
CREATE TABLE realbro_credits (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id VARCHAR(10),
  amount INT,
  type ENUM('PURCHASED', 'USED', 'FREE', 'REFUNDED'),
  description TEXT,
  price_paid INT,
  transaction_id VARCHAR(100),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES realbro_users(id)
);
