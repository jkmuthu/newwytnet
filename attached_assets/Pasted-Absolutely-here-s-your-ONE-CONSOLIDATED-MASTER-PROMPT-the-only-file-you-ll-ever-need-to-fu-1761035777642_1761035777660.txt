## ⚙️ Core Concept

WytNet is powered by a **central Engine** that creates, connects, and controls all components —
**Modules → Apps → Hubs**.

* **Entity:** The smallest data or function unit.
* **Module:** A group of related entities (e.g., Auth, Wallet, Stream).
* **App:** A complete feature or tool made of multiple modules (e.g., WytWall, WytPoints).
* **Hub:** A self-operating portal (e.g., WytNet.com, OwnerNet, VoterNet) built using WytPass Auth and WytEngine Modules.

Each Hub runs independently yet shares one universal **WytPass login** and **WytPoints economy**.

## ⚙️ WytNet Engine – Foundation Logic

The Engine is the **heart of WytNet**, acting as a living framework:

* Auto-registers all modules, apps, and hubs
* Provides APIs and shared services
* Controls authentication (WytPass), points, wallet, and analytics
* Powers the internal AI (WytAI Agent)

Each component includes a **manifest.json**:

```json
{
  "name": "WytWall",
  "type": "core",
  "version": "1.0.0",
  "enabled": true,
  "dependencies": ["auth", "points"],
  "permissions": ["user:read", "stream:write"],
  "routes": ["/wytwall", "/api/wytwall"],
  "description": "Social marketplace for Needs & Offers"
}
```

The Engine must scan all manifests, auto-register them, manage permissions, and generate internal docs.
